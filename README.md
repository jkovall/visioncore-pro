# VisionCore Pro — Курсова Робота

Вебдодаток для інтелектуального візуального пошуку та класифікації товарів за завантаженою фотографією на базі сучасних методів комп'ютерного зору та NoSQL-рішень. Проєкт демонструє розробку асинхронної архітектури для обробки багатовимірних векторів ознак (ембедінгів) та знаходження схожих позицій у режимі реального часу.

## Опис Проєкту

**VisionCore Pro** — це вебплатформа, яка дозволяє користувачам:
- Завантажувати фотографії товарів через адаптивний інтерфейс користувача.
- Автоматично аналізувати візуальний контент і вилучати вектори ознак за допомогою глибокого навчання.
- Знаходити максимально схожі товари, проіндексовані з популярних світових платформ (Amazon, eBay, AliExpress), на основі математичного розрахунку близькості у багатовимірному просторі.
- Використовувати семантичне уточнення запиту за допомогою додаткових текстових підказок.
- Керувати обліковим записом через інтегровану систему автентифікації.

## Основні Функції

### Автентифікація та інтерфейс користувача
- Реєстрація нових користувачів та вхід до системи.
- Адаптивна верстка за сучасними стандартами HTML5 та CSS3 (Grid/Flexbox) для коректного відображення на смартфонах та десктопах.
- Інтерактивна форма завантаження з підтримкою перетягування файлів (Drag-and-Drop).

### Обробка зображень та векторний пошук
- Підтримка валідації форматів і розмірів завантажених графічних файлів (максимум 10MB).
- Вилучення «візуального відбитка» (embedding) за допомогою згорткових нейронних мереж (CNN) та Vision Transformers (ViT).
- Обчислення релевантності видачі на основі метрики косинусної подібності (Cosine Similarity).
- Висока швидкість відповіді (в межах 200–500 мс на запит) завдяки вбудованому векторному індексу MongoDB (`vectorSearch`) та методу K-найближчих сусідів (k-NN).

## Архітектура Проєкту

Проєкт реалізовано за клієнт-серверною архітектурою з чітким розподілом на рівень представлення (Frontend), рівень бізнес-логіки (Backend) та рівень доступу до даних.

### Структура Директорій

```text
VisionCore2.0/
│
├── app/
│   ├── __init__.py                 # Ініціалізація додатка
│   ├── main.py                     # Основна конфігурація FastAPI
│   ├── database.py                 # Підключення до MongoDB через асинхронний драйвер Motor
│   ├── models.py                   # Схеми даних та валідація Pydantic
│   ├── security.py                 # Забезпечення безпеки та автентифікації
│   │
│   ├── routers/
│   │   ├── auth.py                 # Маршрути реєстрації та входу
│   │   ├── upload.py               # Маршрути для обробки завантажених фото
│   │   └── search.py               # Обробка пошукових запитів та ранжування
│   │
│   └── services/
│       └── product_classifier.py   # Модуль аналізу комп'ютерного зору та класифікації
│
├── static/
│   ├── index.html                  # Головна сторінка інтерфейсу (Jinja2)
│   ├── app.js                      # Клієнтська логіка на JavaScript
│   └── styles.css                  # Каскадні таблиці стилів
│
├── uploads/                        # Локальне сховище графічних файлів
│
├── run.py                          # Точка входу та автоматичний пошук вільних портів
├── requirements.txt                # Залежності проєкту
├── .env                            # Змінні оточення
└── README.md                       
```

### Технологічний Стек

- **Backend:** FastAPI (асинхронна обробка запитів на базі стандарту ASGI), Uvicorn, Pydantic.
- **База даних:** MongoDB (NoSQL документоорієнтована модель) + асинхронний драйвер Motor.
- **Аналіз даних та ML:** Концептуальні підходи моделей CLIP, згорткових нейронних мереж (CNN) та Vision Transformers (ViT) для генерації ембедінгів.
- **Обробка зображень:** Бібліотеки комп'ютерного зору (Pillow, NumPy, OpenCV).
- **Frontend:** HTML5, CSS3 (Grid/Flexbox), шаблонізатор Jinja2, Vanilla JavaScript.

## Встановлення та Запуск

### Конфігурація середовища
Створіть файл `.env` у кореневій директорії проєкту та додайте такі параметри:

```env
MONGODB_URL=mongodb://localhost:27017
MONGODB_DB_NAME=visioncore_db
VISIONCORE_HOST=127.0.0.1
VISIONCORE_PORT=8501
```

### Запуск сервера
У файлі `run.py` реалізовано низькорівневий алгоритм автоматичного пошуку вільних мережевих портів (у діапазоні 8000–8100) за допомогою бібліотеки `socket`, що запобігає конфліктам взаємодії.

```bash
python run.py
```
Після запуска вебдодаток автоматично обере доступний порт і буде відкритий для роботи за локальною адресою.
